# 舆论透镜（OpinionLens）产品介绍

## 一、产品定位与愿景

### 1.1 产品定位

**舆论透镜**是一款面向研究者、媒体机构、政府部门和企业的**通用化社交媒体舆论分析平台**。它将复杂的网络科学与自然语言处理技术封装为易用的可视化界面，使用户无需掌握编程技能，即可对任意话题进行深度舆论分析。

> **核心理念**：让每个人都拥有一双洞察舆论的"透视眼"

### 1.2 一句话说明

将海量的社交媒体数据转化为**可行动的洞察**——告诉你谁在说话、说了什么、传播了什么观点、谁在推动舆论。

## 二、实战案例：一次真实的舆论分析全过程

> 以下数据来自我们对**日本政治家高市早苗**相关推文的实际分析，覆盖 213 天、27 万条推文、5 万用户。

### 2.1 你能看到什么？（分析成果总览）

用舆论透镜分析高市早苗话题，我们得到了以下关键发现：

| 维度 | 数据成果 | 业务含义 |
|------|---------|---------|
| **传播规模** | 271,852 条推文 · 50,466 人参与 · 213 天持续讨论 | 话题具有长期社会关注度 |
| **网络结构** | 4 万+节点、12 万+传播关系、最大连通分量占 89% | 信息可有效扩散至大部分用户 |
| **情感分布** | 正面 16.9% · 中立 78% · 负面 5.1%，正负比 3.3:1 | 公众以理性观望为主，支持声大于反对声 |
| **意见领袖** | @takaichi_sanae PageRank = 0.0226，是第二名的 12 倍 | 政治人物本人是绝对传播核心 |
| **话题焦点** | 统一教会相关话题超 8,800 次，成为第二大争议焦点 | 揭示舆论背后的深层议题 |
| **热度峰值** | 2025-10-04 单日互动量突破 1070 万 | 事件驱动型传播，锁定关键时间节点 |
| **异常检测** | 识别多个热度突增事件、可疑账号、协同传播行为 | 揭示非自然传播模式 |

### 2.2 一、如何发现意见领袖？

在传统方法中，你需要逐一查看谁粉丝多、谁转发多。但在舆论透镜中，我们通过**综合影响力得分**自动排名：

```
InfluenceScore = 0.4 × PageRank + 0.3 × Betweenness + 0.3 × DegreeCentrality
```

**实际分析结果：**

| 排名 | 用户 | 粉丝数 | 影响力得分 | 角色定位 |
|------|------|--------|-----------|---------|
| #1 | @takaichi_sanae（本人） | 288万 | 0.0177 | **传播核心** — 发布源头 |
| #2 | @ashitawawatashi | 2.9万 | 0.0091 | **高互动草根** — 持续评论放大声量 |
| #3 | @47news_official | 8.5万 | 0.0079 | **媒体中介** — 信息分发节点 |
| #4 | @iloveyoulove777 | 3万 | 0.0063 | 高互动草根 |
| #5 | @Sankei_news | 87万 | 0.0060 | 媒体中介 |
| … | … | … | … | … |

**发现**：网络呈"**核心-中介-边缘**"三层结构——@takaichi_sanae 本人是绝对核心，@47news/Sankei_news 等媒体账号构成中介层，普通高互动用户构成边缘放大层。✅ 一张排名表，揭示了整个舆论的权力结构。

### 2.3 二、如何量化情感倾向？

传统做法是人工抽样阅读，但在舆论透镜中，我们用**多语言情感词典**对全部 27 万条推文进行了自动分类：

**实际分析结果：**

- **中立 78%**：大多数用户在"收集信息、形成判断"，尚未进入态度固化阶段
- **正面 16.9%** vs **负面 5.1%**，正负比 **3.3 : 1**，支持声量远超反对
- 批评性标签（"最悪の总理大臣"+"やめろ"+"国難"等）合计超 2.4 万次

**⚠️ 异常发现**：虽然总体正负比有利，但批评声音集中在特定标签（#高市やめろ、#高市は統一教会），呈现"反对声音高度标签化、组织化"的结构性特征——这是情感分析揭示的潜在风险信号。

✅ 一张情感分布图 + 一条倾向曲线，让你快速把握舆论基本盘，并发现表面数据下的深层结构。

### 2.5 四、如何发现异常与风险？

舆论透镜内置**异常检测引擎**，自动识别三类异常：

**① 热度突增检测**：自动标记 Z-score > 3 的异常热度日

| 日期 | 推文数 | 变化率 | 可能原因 |
|------|--------|-------|---------|
| 2025-10-04 | 6,832 | +347% | 重大政治事件 |
| 2026-02-01 | 4,615 | +189% | 自民党总裁选举 |
| 2026-02-08 | 3,534 | +112% | 选举相关 |

**② 可疑账号风险评分**：对每个账号打分 0-1，发现高风险账号

- 高发帖量账号（>100条/天）
- 粉丝/发帖比异常（<1）
- 回复比例异常（>90%被动传播）
- 低互动率（发帖多但无人理）

**③ 协同传播识别**：检测"短时间多账号发相似内容"的组织性行为

✅ 异常告警 + 风险评分，让你不再被动应对舆论危机，而是主动预判。

| 平均聚类系数 | 0.0303 | 跨越群体传播效率高 |

✅ 一张力导向网络图 + 社区划分，直观展示：信息从谁传出？经过哪些节点？最终扩散到哪些群体？

### 3.2 功能模块详解

#### 模块一：数据源管理

支持多平台数据源的统一接入与管理：

- **Twitter/X**：通过公开 API 抓取指定关键词/用户/话题的推文数据
- **Reddit**：抓取指定 Subreddit 和关键词下的帖子与评论
- **新闻 RSS**：聚合多个新闻源的 RSS 订阅流
- **YouTube**：抓取视频评论（按热度/时间排序）
- **微博**：抓取指定话题/关键词下的微博及评论
- **自定义导入**：支持 CSV/JSONL 格式数据直接上传

每个数据源均支持：时间范围筛选、语言筛选、数据量上限设置、自动去重与清洗。

#### 模块二：传播网络分析

基于 NetworkX 图论库构建传播有向图，核心算法：

| 指标 | 算法 | 业务含义 |
|------|------|---------|
| PageRank | 迭代随机游走 | 综合网络影响力，考虑间接传播路径 |
| 介数中心性 | Brandes 算法 | 信息桥梁节点，控制跨社区传播 |
| 度中心性 | 直接统计 | 直接互动能力（入度=被传播，出度=主动传播） |
| 聚类系数 | 三元组统计 | 信息封闭程度，小团体识别 |
| 社区检测 | Louvain 算法 | 自动划分观点社群 |

**输出**：交互式力导向网络图（节点大小=影响力，可拖拽、缩放、筛选）、网络统计面板、社区检测结果、传播链路追踪。

#### 模块三：意见领袖识别

基于综合影响力得分的排名系统：

```
InfluenceScore = 0.4 × PageRank + 0.3 × Betweenness + 0.3 × DegreeCentrality
```

**识别结果**：TOP N 意见领袖榜单、用户画像卡片（粉丝数、发推数、影响力得分、所在社区）、多维度对比柱状图、用户详情页。

**价值**：帮助快速定位——谁在主导讨论？谁在推动信息扩散？谁是潜在风险来源？（见上方实战案例）

#### 模块四：情感分析

基于多语言情感词典的自动化分类，支持中文（简体/繁体）、日语、英语等主流语言。

| 类别 | 判定标准 | 典型场景 |
|------|---------|---------|
| 正面（Positive） | 包含支持、赞扬、认同、期待等正向词汇 | "支持！""加油""写得真好" |
| 负面（Negative） | 包含批评、反对、讽刺、恐慌等负向词汇 | "失望""反对""令人担忧" |
| 中立（Neutral） | 无明显情感倾向或为客观陈述 | 新闻转发、事实陈述 |

**分析维度**：全局分布（饼图）、情感倾向指数（0-100 分仪表盘）、情感时间序列、高强度情感识别、正负情感比值。

#### 模块五：话题聚类与追踪

**话题发现**：Hashtag 频率统计 TOP N 排行、基于 Jieba 分词的中文关键词提取、文本聚类。

**话题追踪**：时间轴话题演化图（哪些话题在上升/衰退）、话题生命周期分析（爆发期→扩散期→消退期）、话题关联分析。

#### 模块六：时间序列分析

- 日/周/月三级时间粒度切换
- 舆论热度曲线（发推量、互动量）
- 高热度事件自动检测（Burst Detection），计算变化率
- 周内/日内活跃度分布

#### 模块七：异常检测与风险预警

**热度突增检测**：基于均值+标准差的异常点检测，动态阈值自适应调整。

**协同传播识别**：多账号短时间窗口内发布高度相似内容，输出协同事件列表 + 参与账号 + 内容摘要。

**可疑账号风险评分**：

| 评分维度 | 权重 |
|---------|------|
| 回复比例过高（被动传播特征） | 30% |
| 粉丝/发帖比异常（僵尸粉特征） | 25% |
| 高发帖量（自动化特征） | 20% |
| 日均发帖异常 | 20% |
| 互动率低 | 15% |

风险分级：高风险（≥0.6）、中风险（0.3-0.6）、低风险（<0.3）。

#### 模块八：话题深度搜索

输入任意关键词（人名、事件、政策等），系统生成该话题的专项分析报告：

- 相关推文总数、参与用户数
- 情感分布图、每日讨论量趋势
- 热门参与者（TOP 10）
- 代表性推文（按互动量排序，点击查看全文）

4. **低成本**：纯前端架构，零服务器成本，适合个人研究者和小团队

## 八、使用指南

### 8.1 快速开始

**方式一：在线演示**

访问 https://deft-cobbler-20d1c4.netlify.app 查看已部署的演示系统（内置高市早苗数据集，含 27 万条推文、5 万用户、213 天完整分析结果）

**方式二：本地部署**

```bash
# 1. 克隆项目
git clone https://github.com/LeonTang-LeonTang/OpinionLens.git
cd OpinionLens

# 2. 安装前端依赖
cd frontend && npm install

# 3. 启动开发服务器
npm run dev
# 访问 http://localhost:3000

# 4. 准备分析数据（如有原始 JSONL 数据）
python src/data_preprocessing.py --input your_data.jsonl
python src/network_analysis.py
python src/sentiment_topic_analysis.py
python src/build_embedded_data.py

# 5. 重新构建（将数据嵌入 bundle）
npm run build
```

### 8.2 数据准备

**支持的输入格式**：
- Twitter JSONL（每行一条推文，含 id, screen_name, text, created_at, favorite_count, retweet_count 等字段）
- 通用 CSV（含必需列：screen_name, text, created_at）
- 手动在数据源页面选择已有数据集

**数据量建议**：单次分析建议 1,000 - 500,000 条推文，效果最佳
