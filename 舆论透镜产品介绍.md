# 舆论透镜（OpinionLens）产品介绍

## 一、产品定位与愿景

### 1.1 产品定位

**舆论透镜**是一款面向研究者、媒体机构、政府部门和企业的**通用化社交媒体舆论分析平台**。它将复杂的网络科学与自然语言处理技术封装为易用的可视化界面，使用户无需掌握编程技能，即可对任意话题进行深度舆论分析。

> **核心理念**：让每个人都拥有一双洞察舆论的"透视眼"

### 1.2 产品愿景

当前，舆论场已成为社会事件的"第一现场"——重大政策出台、突发事件、名人事件，舆论往往在数小时内完成形成、扩散与定性的完整生命周期。传统的人工监测方式效率低下，且难以把握宏观规律。

舆论透镜的愿景是：**将海量的社交媒体数据转化为可行动的洞察**，帮助用户在信息洪流中看清舆论的本质结构。

### 1.3 目标用户

| 用户类型 | 核心需求 | 典型场景 |
|---------|---------|---------|
| **学术研究者** | 舆情数据量化分析、传播机制验证 | 论文数据支撑、课题研究 |
| **媒体记者** | 快速把握事件全貌、追踪舆论走向 | 报道素材准备、深度选题 |
| **政府舆情部门** | 实时监测、异常预警、风险评估 | 舆论危机响应、民意收集 |
| **企业品牌/公关** | 竞品舆情监控、口碑分析 | 危机预警、声誉管理 |
| **政治人物团队** | 公众态度追踪、传播效果评估 | 竞选策略、形象管理 |

### 2.2 功能模块详解

#### 模块一：数据源管理

支持多平台数据源的统一接入与管理：

- **Twitter/X**：通过公开 API 抓取指定关键词/用户/话题的推文数据
- **Reddit**：抓取指定 Subreddit 和关键词下的帖子与评论
- **新闻 RSS**：聚合多个新闻源的 RSS 订阅流
- **YouTube**：抓取视频评论（按热度/时间排序）
- **微博**：抓取指定话题/关键词下的微博及评论
- **自定义导入**：支持 CSV/JSONL 格式数据直接上传

每个数据源均支持：
- 时间范围筛选（起止日期、相对时间如"最近7天"）
- 语言筛选
- 数据量上限设置
- 自动去重与清洗

#### 模块二：传播网络分析

核心算法层，基于 NetworkX 图论库构建：

**网络构建**：
```
用户节点 → 回复/引用关系 → 信息传播有向图
边权重 = 互动次数（点赞+转发+回复的综合）
```

**核心指标计算**：

| 指标 | 算法 | 业务含义 |
|------|------|---------|
| PageRank | 迭代随机游走 | 综合网络影响力，考虑间接传播路径 |
| 介数中心性 | Brandes 算法 | 信息桥梁节点，控制跨社区传播 |
| 度中心性 | 直接统计 | 直接互动能力（入度=被传播，出度=主动传播） |
| 聚类系数 | 三元组统计 | 信息封闭程度，小团体识别 |
| 连通分量 | BFS/DFS | 社区划分基础，衡量信息可达性 |

**输出**：
- 交互式力导向网络图（节点大小=影响力，可拖拽、缩放、筛选）
- 网络统计面板（节点数、边数、密度、平均路径长度）
- 社区检测结果（不同颜色区分社区，显示核心成员）
- 传播链路追踪（从源帖到扩散末端的完整路径）

#### 模块三：意见领袖识别

基于综合影响力得分的排名系统：

```
InfluenceScore = 0.4 × PageRank + 0.3 × Betweenness + 0.3 × DegreeCentrality
```

**识别结果**：
- TOP N 意见领袖榜单（可配置排名数量）
- 用户画像卡片：粉丝数、发推数、影响力得分、所在社区
- 多维度对比柱状图（PageRank / 介数 / 度中心性三轴）
- 用户详情页：历史发帖、互动热力图、传播路径

**价值**：帮助用户快速定位舆论场中的关键节点——谁在主导讨论？谁在推动信息扩散？谁是潜在的风险来源？

#### 模块四：情感分析

基于多语言情感词典的自动化分类：

**支持语言**：中文（简体/繁体）、日语、英语、韩语、法语等主流语言

**分类体系**：

| 类别 | 判定标准 | 典型场景 |
|------|---------|---------|
| 正面（Positive） | 包含支持、赞扬、认同、期待等正向词汇 | "支持！""加油""写得真好""值得关注" |
| 负面（Negative） | 包含批评、反对、讽刺、恐慌等负向词汇 | "失望""反对""令人担忧""简直是笑话" |
| 中立（Neutral） | 无明显情感倾向或为客观陈述 | 新闻转发、事实陈述、技术讨论 |

**分析维度**：
- 全局分布（饼图）
- 情感倾向指数（0-100 分的仪表盘）
- 情感时间序列（情感随时间的变化趋势）
- 高强度情感识别（区分"一般负面"和"强烈负面"）
- 正负情感比值（舆论支持基础指标）

#### 模块五：话题聚类与追踪

**话题发现**：
- Hashtag（话题标签）频率统计，TOP N 排行
- 关键词提取（基于 TF-IDF 和 Jieba 分词的中文处理）
- 文本聚类（K-Means 或 LDA 主题模型）

**话题追踪**：
- 时间轴话题演化图（哪些话题在上升，哪些在衰退）
- 话题生命周期分析（爆发期→扩散期→消退期）
- 话题关联分析（哪些话题经常同时出现）

**输出**：
- 话题云（字体大小=出现频率）
- 月度话题变化热力图
- 话题对比分析表

#### 模块六：时间序列分析

**分析维度**：
- 日/周/月三级时间粒度切换
- 舆论热度曲线（发推量、互动量）
- 周内分布（周一至周日活跃度对比）
- 事件关联（高热度日与已知事件的时间对齐）

**高热度事件识别**：
- 自动检测热度突增点（Burst Detection）
- 计算变化率（"+X%"）
- 关联分析（同一天内是否有多个相关话题同时上升）

#### 模块七：异常检测与风险预警

**热度突增检测**：
- 统计方法：基于均值+标准差的异常点检测
- 动态阈值：根据历史基线自适应调整

**协同传播识别**（Coordinated Behavior Detection）：
- 识别特征：多账号在短时间窗口内发布高度相似内容
- 检测维度：文本相似度（编辑距离/Jaccard）、时间窗口、账号特征
- 输出：协同事件列表，包含参与账号、时间窗口、内容摘要

**可疑账号风险评分**：
- 评分维度：
  - 发帖频率异常（过高或过低）
  - 粉丝/发帖比异常（僵尸粉特征）
  - 回复比例异常（被动传播账号特征）
  - 账户年龄与发帖量不匹配
- 风险分级：高风险（≥0.6）、中风险（0.3-0.6）、低风险（<0.3）
- 输出：高/中/低风险账号列表 + 详细风险因素说明

#### 模块八：话题深度搜索

**功能**：
- 输入任意关键词（人名、事件、政策等）
- 系统在已加载数据集中搜索相关推文
- 生成该话题的专项分析报告

**输出**：
- 话题相关推文总数、参与用户数
- 情感分布图
- 每日讨论量趋势
- 热门参与者（TOP 10）
- 代表性推文（按互动量排序，点击查看全文）

## 四、数据分析流程

### 4.1 端到端分析流水线

```
第1步：数据采集
───────────────
输入：关键词/话题 → Twitter API / Reddit API / RSS → 原始 JSONL
         │
         ▼
第2步：数据清洗
───────────────
无效记录过滤 │ 文本规范化 │ 日期标准化 │ 字段提取
         │
         ▼
第3步：特征工程
───────────────
参与度得分计算 │ 时间特征提取 │ 情感词匹配 │ 话题标签提取
         │
         ▼
第4步：网络构建
───────────────
回复/引用关系 → 有向加权图（NetworkX）→ 指标计算
(PageRank / 介数 / 度中心性 / 聚类系数)
         │
         ▼
第5步：分析输出
───────────────
JSON 文件生成 → Vite 打包嵌入 → 前端可视化
```

### 4.2 典型分析周期

| 阶段 | 耗时（参考） | 说明 |
|------|------------|------|
| 数据采集 | 5-30 分钟 | 取决于数据量和平台限制 |
| 数据清洗 | 1-5 分钟 | 自动完成，含去重和规范化 |
| 网络构建 | 5-20 分钟 | 10万级节点/边约需10分钟 |
| PageRank 计算 | 1-3 分钟 | 迭代算法，节点越多越慢 |
| 情感分析 | 2-5 分钟 | 词典匹配，速度快 |
| 社区检测 | 3-10 分钟 | Louvain 算法 |
| 可视化生成 | <1 分钟 | 静态 JSON 生成 |
| **总计** | **约 20-70 分钟** | 从原始数据到在线可查看 |

## 六、应用场景案例

### 6.1 案例一：政治人物公众形象分析

**场景**：某政治人物团队希望了解其在社交媒体上的公众形象和传播结构。

**操作**：
1. 采集该人物相关推文（关键词 = 人物姓名）
2. 等待分析完成（20-60 分钟）
3. 查看 Dashboard 概览
4. 进入"意见领袖"模块，查看谁在主导讨论
5. 进入"情感分析"，了解支持/反对比例
6. 进入"话题聚类"，发现核心争议话题

**输出**：一份包含传播网络图、TOP 影响力账号、情感分布、热门话题的分析报告，可直接用于策略制定。

### 6.2 案例二：突发事件舆论追踪

**场景**：某企业遭遇公关危机，需要实时追踪舆论走向。

**操作**：
1. 设置数据采集任务（关键词 = 企业名/事件名）
2. 每小时自动抓取新数据
3. 定期刷新分析结果
4. 观察"异常检测"模块的热度突增告警
5. 追踪"协同传播"事件，识别是否有组织性攻击

**输出**：实时舆论热力图、异常事件告警、可疑传播行为报告。

### 6.3 案例三：学术研究数据支撑

**场景**：研究者需要验证"社交媒体舆论传播是否具有小世界特性"。

**操作**：
1. 导入目标话题数据
2. 查看传播网络统计（密度、平均路径长度、聚类系数）
3. 绘制网络图，验证结构特征
4. 导出数据用于进一步统计分析

**输出**：网络结构指标、可视化图谱，可作为论文数据支撑。

## 八、使用指南

### 8.1 快速开始

**方式一：在线演示**
访问 https://deft-cobbler-20d1c4.netlify.app 查看已部署的演示系统（内置高市早苗数据集）

**方式二：本地部署**

```bash
# 1. 克隆项目
git clone https://github.com/LeonTang-LeonTang/OpinionLens.git
cd OpinionLens

# 2. 安装前端依赖
cd frontend && npm install

# 3. 启动开发服务器
npm run dev
# 访问 http://localhost:3000

# 4. 准备分析数据（如有原始 JSONL 数据）
python src/data_preprocessing.py --input your_data.jsonl
python src/network_analysis.py
python src/sentiment_topic_analysis.py
python src/build_embedded_data.py

# 5. 重新构建（将数据嵌入 bundle）
npm run build
```

### 8.2 数据准备

**支持的输入格式**：
- Twitter JSONL（每行一条推文，含 id, screen_name, text, created_at, favorite_count, retweet_count, in_reply_to_screen_name 等字段）
- 通用 CSV（含必需列：screen_name, text, created_at）
- 手动在数据源页面选择已有数据集

**数据量建议**：单次分析建议 1,000 - 500,000 条推文，效果最佳
